# Структура проекту Xeno RAT

Цей документ описує структуру каталогів та файлів проекту Xeno RAT, а також їх призначення.

## Коренева директорія

| Файл/Директорія     | Опис                                                                                             |
| ------------------- | ---------------------------------------------------------------------------------------------------- |
| `/.gitignore`       | Визначає файли та директорії, які повинні бути проігноровані системою контролю версій Git.      |
| `/LICENSE`          | Файл ліцензії (MIT License), що визначає умови використання та розповсюдження вихідного коду.    |
| `/Plugins/`         | Директорія, що містить вихідний код усіх плагінів, що розширюють функціонал RAT.                     |
| `/README.md`        | Основний файл з описом проекту, його можливостями, інструкціями з використання та правовою інформацією. |
| `/logo.png`         | Логотип проекту Xeno RAT.                                                                            |
| `/xeno rat client/` | Вихідний код клієнтської частини RAT, яка встановлюється на цільову машину.                        |
| `/xeno rat server/` | Вихідний код серверної частини RAT (панель управління) з графічним інтерфейсом для оператора.      |

## Директорія `xeno rat client`

Клієнтська частина, призначена для виконання на віддаленій машині.

| Файл                | Опис                                                                                                |
| ------------------- | ------------------------------------------------------------------------------------------------------- |
| `Program.cs`        | Основна точка входу клієнта. Керує підключенням до сервера, персистентністю та обробкою винятків. |
| `Node.cs`           | Реалізує вузол зв'язку, що керує основним та дочірніми сокетами для різних завдань.                    |
| `Handler.cs`        | Обробляє команди, що надходять від сервера, та ініціює відповідні дії (наприклад, запуск плагінів). |
| `SocketHandler.cs`  | Низькорівневий обробник для надсилання та отримання даних через сокети, включаючи шифрування та стиснення.     |
| `Encryption.cs`     | Реалізує шифрування (AES) для захисту комунікацій між клієнтом та сервером.                           |
| `Compression.cs`    | Реалізує стиснення даних (LZMA) для зменшення мережевого трафіку.                                        |
| `DllHandler.cs`     | Відповідає за динамічне завантаження та виконання плагінів з пам'яті.                                     |
| `Utils.cs`          | Містить допоміжні функції: отримання інформації про систему, перевірка прав адміністратора, тощо.    |
| `*.csproj`          | Файл проекту C#, що містить конфігурацію збірки для клієнтського додатку.                           |

## Директорія `xeno rat server`

Серверна частина (панель управління) з графічним інтерфейсом на Windows Forms.

| Файл/Директорія     | Опис                                                                                                   |
| ------------------- | ---------------------------------------------------------------------------------------------------------- |
| `Program.cs`        | Точка входу для запуску додатку Windows Forms.                                                          |
| `MainForm.cs`       | Основна форма додатку, що відображає список підключених клієнтів та надає доступ до функцій. |
| `Listener.cs`       | Прослуховує вхідні підключення від клієнтів.                                                             |
| `Node.cs`           | Серверна реалізація вузла зв'язку для взаємодії з клієнтами.                                           |
| `Forms/`            | Директорія, що містить UI-форми для взаємодії з різними плагінами (чат, файловий менеджер тощо). |
| `country_flags/`    | Ресурси для відображення прапорів країн на основі геолокації IP-адрес клієнтів.                           |
| `*.csproj`          | Файл проекту C#, що містить конфігурацію збірки для серверного додатку.                                |
| `*.sln`             | Файл рішення Visual Studio, що об'єднує всі проекти (клієнт, сервер, плагіни).                         |

## Директорія `Plugins`

Містить вихідний код для кожного плагіна, що реалізує певний функціонал. Кожен плагін є окремим проектом C#.

| Плагін              | Опис                                                                                             |
| ------------------- | ---------------------------------------------------------------------------------------------------- |
| `Chat`              | Реалізує двосторонній чат між оператором та клієнтом.                                              |
| `File manager`      | Надає функціонал для управління файловою системою віддаленої машини.                           |
| `Fun`               | Містить "розважальні" функції, такі як управління лотком CD/DVD та вимкнення/ввімкнення монітора. |
| `HiddenBrowser`     | Запускає прихований (невидимий) браузер на машині клієнта.                                               |
| `Hvnc`              | Реалізує Hidden VNC, дозволяючи створити прихований віртуальний робочий стіл для непомітного управління.   |
| `KeyLogger`         | Перехоплює та надсилає натискання клавіш у реальному часі.                                           |
| `KeyLoggerOffline`  | Зберігає натискання клавіш локально, коли клієнт не в мережі, та надсилає їх при підключенні.          |
| `LiveMicrophone`    | Дозволяє прослуховувати мікрофон клієнта в реальному часі.                                            |
| `PassGrab`          | Витягує збережені паролі, cookie та іншу чутливу інформацію з браузерів та додатків.     |
| `ProcessManager`    | Дозволяє переглядати та керувати запущеними процесами на віддаленій машині.                       |
| `Registry Manager`  | Надає інтерфейс для перегляду та редагування реєстру Windows.                                |
| `ReverseProxy`      | Реалізує зворотний SOCKS5 проксі, дозволяючи тунелювати трафік через машину клієнта.                 |
| `ScreenControl`     | Забезпечує віддалений перегляд та управління робочим столом клієнта.                                   |
| `Shell`             | Надає віддалений доступ до командного рядка (cmd.exe) на машині клієнта.                        |
| `Startup`           | Керує елементами автозавантаження для забезпечення персистентності.                                     |
| `System`            | Дозволяє виконувати системні дії, такі як вимкнення або перезавантаження комп'ютера.              |
| `Uacbypass`         | Містить методи для обходу контролю облікових записів користувачів (UAC) у Windows.                     |
| `WebCam`            | Дозволяє переглядати зображення з веб-камери клієнта в реальному часі.                           |
